## 1. 通用规范

### 1.1 响应格式

所有接口均返回统一格式：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

**失败响应：**

```json
{
  "code": 400,
  "message": "邮箱格式不正确",
  "data": null
}
```

### 1.2 分页格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [ ... ],
    "total": 100,
    "page": 1,
    "pageSize": 10,
    "totalPages": 10
  }
}
```

### 1.3 认证方式

管理员接口需要在请求头携带 JWT Token：

```
Authorization: Bearer <token>
```

### 1.4 请求头

```
Content-Type: application/json
Accept-Language: zh-CN
```

## 2. 询价接口

### 2.1 提交询价

**前端联系页面的表单提交接口（最重要的接口）**

```
POST /inquiries
```

**请求体：**

```json
{
  "name": "张三",
  "company": "某某机械有限公司",
  "email": "zhangsan@example.com",
  "phone": "+86 13800138000",
  "product_category": "灰铸铁件",
  "quantity": "500件",
  "message": "需要HT200材质，重量约20kg，用于机床底座，需要加工精度±0.1mm，请提供报价。"
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 长度限制 | 说明 |
|------|------|------|----------|------|
| name | string | ✅ | 2-100 | 客户姓名 |
| company | string | ❌ | 0-200 | 公司名称 |
| email | string | ✅ | 5-200 | 邮箱地址（需验证格式） |
| phone | string | ❌ | 0-50 | 联系电话 |
| product_category | string | ❌ | 0-100 | 产品类别 |
| quantity | string | ❌ | 0-100 | 预计采购量 |
| message | string | ✅ | 10-2000 | 详细需求描述 |

**成功响应：**

```json
{
  "code": 200,
  "message": "询价已提交，我们将在1个工作日内与您联系",
  "data": {
    "inquiry_id": "INQ-2024-0001",
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

**失败响应示例：**

```json
{
  "code": 422,
  "message": "邮箱格式不正确",
  "data": {
    "errors": [
      { "field": "email", "message": "邮箱格式不正确" }
    ]
  }
}
```

**后端处理逻辑：**
1. 验证必填字段和格式
2. 防刷检查：同一IP 10分钟内最多提交3次
3. 将数据存入 `inquiries` 表
4. 异步发送邮件通知给销售团队
5. 异步发送确认邮件给客户
6. 返回成功响应

**邮件通知模板（销售团队）：**
```
主题：[新询价] {客户姓名} - {产品类别} - {时间}
内容：
  姓名：{name}
  公司：{company}
  邮箱：{email}
  电话：{phone}
  产品：{product_category}
  数量：{quantity}
  需求：{message}
  
  [点击处理此询价] → 管理后台链接
```

### 2.2 获取询价编号状态（可选）

客户提交后可通过此接口查询处理状态。

```
GET /inquiries/{inquiry_id}/status
```

**路径参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| inquiry_id | string | 询价编号，如 INQ-2024-0001 |

**响应：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "inquiry_id": "INQ-2024-0001",
    "status": "processing",
    "status_label": "处理中",
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

**状态枚举：**

| 值 | 说明 |
|----|------|
| pending | 待处理 |
| processing | 处理中 |
| replied | 已回复 |
| closed | 已关闭 |

## 5. 文件上传接口

### 5.1 上传图片

**管理员上传产品图片使用（需要认证）**

```
POST /upload/image
Authorization: Bearer <token>
Content-Type: multipart/form-data
```

**请求体（form-data）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | ✅ | 图片文件 |
| folder | string | ❌ | 存储目录，如 products / hero |

**限制：**
- 文件格式：JPG、PNG、WebP
- 文件大小：最大 5MB
- 建议尺寸：见素材说明文档

**响应：**

```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "url": "https://your-cdn.com/products/1705123456789.jpg",
    "filename": "1705123456789.jpg",
    "size": 245678,
    "width": 800,
    "height": 600
  }
}
```

**后端处理：**
1. 验证文件类型（检查 MIME type，而非仅扩展名）
2. 验证文件大小
3. 重命名为时间戳+随机字符串（防止路径遍历）
4. 上传至阿里云 OSS / 腾讯云 COS
5. 返回 CDN URL

## 6. 管理员认证接口

### 6.1 管理员登录

```
POST /admin/login
```

**请求体：**

```json
{
  "username": "admin",
  "password": "your_password"
}
```

**成功响应：**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 86400,
    "admin": {
      "id": 1,
      "username": "admin",
      "role": "super_admin"
    }
  }
}
```

**失败响应：**

```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "data": null
}
```

**安全要求：**
- 密码必须使用 bcrypt 加密存储（salt rounds ≥ 10）
- 登录失败5次锁定账号30分钟
- Token 有效期 24 小时
- 支持 Token 刷新

---

## 9. 错误码说明

| HTTP状态码 | code | 说明 |
|-----------|------|------|
| 200 | 200 | 成功 |
| 400 | 400 | 请求参数错误 |
| 401 | 401 | 未认证或 Token 已过期 |
| 403 | 403 | 权限不足 |

## 10. 数据库设计

### 10.1 表结构（MySQL）

#### inquiries 询价表

```sql
CREATE TABLE inquiries (
  id          INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  inquiry_id  VARCHAR(20) UNIQUE NOT NULL COMMENT '询价编号 INQ-YYYY-XXXX',
  name        VARCHAR(100) NOT NULL COMMENT '客户姓名',
  company     VARCHAR(200) DEFAULT NULL COMMENT '公司名称',
  email       VARCHAR(200) NOT NULL COMMENT '邮箱',
  phone       VARCHAR(50) DEFAULT NULL COMMENT '电话',
  product_category VARCHAR(100) DEFAULT NULL COMMENT '产品类别',
  quantity    VARCHAR(100) DEFAULT NULL COMMENT '预计采购量',
  message     TEXT NOT NULL COMMENT '详细需求',
  status      ENUM('pending','processing','replied','closed') NOT NULL DEFAULT 'pending',
  source      VARCHAR(50) DEFAULT 'website' COMMENT '来源渠道',
  ip_address  VARCHAR(50) DEFAULT NULL,
  admin_note  TEXT DEFAULT NULL COMMENT '内部备注',
  replied_at  DATETIME DEFAULT NULL,
  created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_status (status),
  INDEX idx_created_at (created_at),
  INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='询价记录表';
```

#### product_categories 产品分类表

```sql
CREATE TABLE product_categories (
  id          INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  name        VARCHAR(100) NOT NULL COMMENT '分类名称',
  name_en     VARCHAR(100) DEFAULT NULL COMMENT '英文名称',
  sort_order  INT NOT NULL DEFAULT 0,
  created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='产品分类表';
```

#### products 产品表

```sql
CREATE TABLE products (
  id            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  name          VARCHAR(200) NOT NULL COMMENT '产品名称',
  name_en       VARCHAR(200) DEFAULT NULL COMMENT '英文名称',
  category_id   INT UNSIGNED NOT NULL COMMENT '分类ID',
  material      VARCHAR(200) DEFAULT NULL COMMENT '材质规格',
  weight_range  VARCHAR(100) DEFAULT NULL COMMENT '重量范围',
  standard      VARCHAR(100) DEFAULT NULL COMMENT '执行标准',
  description   TEXT DEFAULT NULL COMMENT '产品描述',
  cover_image   VARCHAR(500) DEFAULT NULL COMMENT '封面图URL',
  images        JSON DEFAULT NULL COMMENT '多图URL数组',
  tag           VARCHAR(50) DEFAULT NULL COMMENT '标签',
  is_featured   TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否首页推荐',
  is_active     TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否上架',
  sort_order    INT NOT NULL DEFAULT 0,
  created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (category_id) REFERENCES product_categories(id),
  INDEX idx_category (category_id),
  INDEX idx_featured (is_featured),
  INDEX idx_active (is_active),
  FULLTEXT INDEX ft_search (name, material)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='产品信息表';
```

#### admins 管理员表

```sql
CREATE TABLE admins (
  id          INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
```

## 附录：前端 API 调用示例

在前端代码中，将以下 fetch 示例集成到对应页面：

```typescript
// utils/api.ts
const BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:3000/api/v1';

// 提交询价
export async function submitInquiry(data: InquiryForm) {
  const res = await fetch(`${BASE_URL}/inquiries`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data),
  });
  return res.json();
}

// 获取产品列表
export async function getProducts(params: ProductQuery) {
  const query = new URLSearchParams(params as any).toString();
  const res = await fetch(`${BASE_URL}/products?${query}`);
  return res.json();
}

// 获取分类列表
export async function getCategories() {
  const res = await fetch(`${BASE_URL}/categories`);
  return res.json();
}
```
